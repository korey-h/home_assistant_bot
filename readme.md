# Home Assistant bot
## Описание
1. Телеграм-бот взаимодействует с сервером Home Assistant (HA) посредством API. Функции бота доступны только для пользователей, чей *"telegram id" внесен в файл _"users.txt"_ c присвоенной ролью _"manager"_.
2. Использование бота начинается с отправки ему команды _/start_. Во время обработки этой команды происходит считывание данных из файлов _"users.txt"_ и _"active_devices.txt"_ (список используемых устройств).
В результате в Телеграм возвращается сообщение с интерактивными кнопками, каждая из которых соответствует устройству из перечня _"active_devices.txt"_. Нажатие на такую кнопку вызывает новое сообщение, в котором содержится перечень доступных для устройства действий в виде интерактивных кнопок.
Нажатие на кнопку с названием действия приводит данное действие в исполнение. По итогу его выполгнение пользователю направляется сообщение с состоянием устройства. Например, для реле доступными действиями являются: "Включить", "Отключить" или "переключить", результатами выполнения действия: "Включено", "Отключено" или "Недоступно" (если устройство не подключено к серверу HA).

## Настройка
1. Для работы бота необходимо заполнить три файла с настройками: _".env"_, _"users.txt"_ и _"active_devices.txt"_.
2. В файл ".env" вносятся значения для следующих параметров:
TG_TOKEN - токен доступа к боту, полученный от @BotFather;
DEVELOP_ID - телеграмм-id пользователя, которому будут направляться сообщения об ошибках работы бота;
HOST - адрес сервера НА, например http://127.0.0.1:8123;
HA_TOKEN - токен из раздела "Безопасность" -> "Долгосрочные токены доступа" веб-интерфейса HA.

Пример заполнения:
```
TG_TOKEN = '1000000006:Aaaaaaap65XU9QoyyyyyyyyyyZkNPZT8888'
DEVELOP_ID = '10000000'
HOST = 'http://127.0.0.1:8123'
HA_TOKEN = 'eyJhbGciO....5cCCJ9.eyJpcNTk.zQ...zfQ.-Fj_wck......H7s'
```
3. В файл "users.txt" вносится телеграмм-id пользователей, которым разрешен доступ к боту. Информация вносится в формате JSON.
Пример заполнения:
```
[
{"user_id": "10000000", "role": "manager"}
]
```
4. В файл "active_devices.txt" вносятся данные устройст, с которыми взаимодействует бот. Информация вносится в формате JSON.

Пример заполнения:
```
[
{"entity_id": "switch.zigbee_switch_t1_switch_1", "name": "Реле тестовое", "domain": "switch"}
]
```
Описание ключей:
entity_id - идентификатор объекта. В веб-интерфейсе HA находится после нажатия на значок настройки ("шестеренку") устройства.
name - имя, отображаемое на кнопке.
domain - наименование группы схожих по функциям устройств. По этому значению выполняется поиск доступных для устройства действий. TODO: сделать автоматическое определение домена.
